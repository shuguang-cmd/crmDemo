# CRM 智能办公系统 - 开发文档

本系统旨在提供一个全面的客户关系管理解决方案，涵盖了从营销机会发现到客户服务维护的全生命周期管理。

## 1. 业务功能模块

根据当前前端面板设计，系统主要分为以下五大核心模块：

### 1.1 营销管理
*   **营销机会管理**：记录与潜在客户的初步接触信息，包括机会来源、成功几率、联系人信息等。
*   **客户开发计划**：针对具体的营销机会制定详细的开发步骤，并记录执行结果。

### 1.2 客户管理
*   **客户信息管理**：维护正式客户的详细档案，包括信用等级、满意度、地址等。
*   **客户流失管理**：监控客户活跃度，对于疑似流失客户进行预警及流失原因分析。

### 1.3 服务管理
*   **服务创建**：接待客户反馈，创建服务单。
*   **服务分配**：将服务单指派给相应的客服或技术人员。
*   **服务处理**：记录服务的解决过程。
*   **服务反馈**：收集客户对服务结果的评价。
*   **服务归档**：完成服务流程并转入历史记录。

### 1.4 统计报表
*   **多维度分析**：
    *   客户贡献分析（消费金额、利润贡献等）。
    *   客户构成分析（按等级、来源等统计）。
    *   客户服务分析（服务类型分布）。
    *   客户流失分析。

### 1.5 系统设置
*   **用户管理**：管理系统登录账号及其基本信息。
*   **角色管理**：定义系统角色（如管理员、销售、经理）并分配权限。
*   **菜单管理**：动态配置系统的导航菜单。
*   **字典管理**：维护系统常用的下拉框选项（如服务类型、客户等级等）。

## 3. 开发规范与流程

### 3.1 新增模块流程
1.  **数据库设计**：创建表结构，使用 `generatorConfig.xml` 生成实体类与 Mapper。
2.  **Service 层实现**：继承 `BaseService`，实现特定的业务校验逻辑。
3.  **Controller 层实现**：
    *   继承 `BaseController`。
    *   编写页面跳转方法（返回 `.ftl` 路径）。
    *   编写 API 接口（使用 `@ResponseBody` 返回 `ResultInfo`）。
4.  **前端实现**：
    *   在 `views/` 下创建对应的 FTL 文件。
    *   在 `public/js/` 下创建对应的 JavaScript 脚本，使用 `layui.table` 渲染数据。

### 3.2 权限控制说明
*   系统通过 `Cookie` 存储用户信息（`userIdStr`, `userName`）。
*   后端可通过拦截器或 AOP 对请求进行登录校验与权限鉴权。
